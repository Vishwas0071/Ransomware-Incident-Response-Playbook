# Ransomware Incident Response Playbook

## Overview
This is a complete incident response playbook for ransomware attacks, independently developed by me as part of my cybersecurity portfolio. It follows the NIST SP 800-61 Rev 2 (Computer Security Incident Handling Guide) six-phase framework and covers everything from preparation through to post-incident review.

## NIST IR Phases Covered

### Phase 1 — Preparation
- Required technical and organisational controls
- IR team roles and responsibilities
- Incident severity definitions and SLAs

### Phase 2 — Detection & Analysis
- Ransomware detection indicators
- First 30-minute triage procedure (step by step)
- Ransomware family identification method
- Scope assessment process

### Phase 3 — Containment
- Network isolation procedures (EDR and manual)
- Lateral movement prevention steps
- Evidence preservation methodology
- Chain of custody documentation

### Phase 4 — Eradication
- Root cause analysis methodology
- Initial access vector identification
- Malicious artefact removal
- System rebuilding vs cleaning decision framework

### Phase 5 — Recovery
- Recovery sequence (priority order)
- Backup integrity verification
- Staged return to operations
- Enhanced post-recovery monitoring

### Phase 6 — Post-Incident Activity
- GDPR breach notification requirements (Article 33 — 72 hour rule)
- Data Protection Commission Ireland notification process
- Lessons learned framework
- IR metrics and timeline documentation

## Key Features
- Incident severity matrix (CRITICAL / HIGH / MEDIUM / LOW)
- IR team RACI with escalation paths
- Step-by-step containment decision tree
- GDPR breach notification guidance (Irish context — DPC)
- Ransom payment decision framework
- Key contacts — GNCCB, DPC Ireland, IRISSCERT, nomoreransom.org
